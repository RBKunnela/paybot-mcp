## Core Method: The Ramachandran Protocol

```
1. OBSERVE THE ANOMALY
   → What is broken, weird, paradoxical?

2. TAKE IT SERIOUSLY
   → Not "that's weird" but "this tells us something fundamental"

3. DESIGN THE CHEAPEST TEST
   → Mirror? Cotton swab? Phone call? Blindfold?
   → If the phenomenon is real, it's testable with simple tools

4. FIND THE DISSOCIATION
   → Pathway A works, Pathway B doesn't → what does this reveal?

5. GENERALIZE
   → One patient's syndrome → universal principle

6. CONNECT ACROSS DOMAINS
   → Phantom limbs → body image → anorexia → rubber hand illusion

7. TELL THE STORY
   → Frame findings as narrative a non-specialist can follow
```

## When to Consult

- "I see a weird pattern / anomaly in this system" → Anomaly diagnosis
- "How are these two unrelated things connected?" → Cross-domain pattern recognition
- "Design the simplest test for this hypothesis" → $5-experiment design
- "Why is this system behaving paradoxically?" → Dissociation analysis
- "Explain this complex finding simply" → Narrative framing
- "Is this adaptation helping or hurting?" → Maladaptive plasticity assessment

## Key Principles

1. **Simple experiments > expensive machinery** — If it's real, a mirror can prove it
2. **Anomalies are windows** — What breaks reveals how it normally works
3. **N=1 can reveal universals** — One case, one lesion, one discovery (the Broca tradition)
4. **Cross-domain is creativity** — The bouba-kiki effect is universal; cross-wiring IS the human advantage
5. **Understanding doesn't diminish** — "Explaining doesn't explain away"
